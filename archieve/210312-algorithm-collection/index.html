Logistic regression, ROC & AUC, P-value

Post modified 2021-04-04 | In Video Report

The following are notes when viewing StatQuest’s videos. StatQuest is recommended for anyone like me in need of statistic knowledge but have not taken systematic statistic training.

Intro Machine learning

  • all about making prediction and classification.
  • Training data & testing data - Fit the training well but making poor predictions is called Bias-Variance Tradeoff

Cross Validation

  • allow us to compare different machine learning methods and know how well they are doing
  • first 25% for training, rest testing <- which 25%?
  • use different 25% each time four-fold arbitrary
  • usual: ten-fold

Logistic regression

regression for logistic (boolean)

- probability curve

Selectivity and Specificity

  • Sensitivity(判真) = True Positive / ( True Positives + False Negatives)
  • Specificity(辨假) = False Positive / (False Positives + True Negatives)
  • Precision = True Positive / ( True Positives + False Positives)

*true/false compare actual vs predicted, positive/negative is predicted result

Receiver Operator Characteristic (ROC) & Area under the curve(AUC)

- set a threshold at the logistic regression

- threshold = 0.1 = false positive

- threshold = 0.9 = false negative

- confusion matrix too complicated for comparing different threshold

ROC & AUC

ROC graph = true positive rate (sensitivity) vs false positive rate(=1-specificity)

AUC of ROC larger suggesting the curve better

P-value

  • probability of [generating events of equal or rarer probability]

比如说在一个高斯分布pdf里面，某一个值k对应的p value是 可能性（纵坐标）小于p的两个tail面积和，如果k敲好是mean，那么p value就等于1

Probability vs Likelihood

  • Probabilities are areas under fixed distribution
  • Likelihoods are y-axis values for fixed data pointes with distributions that can be move

Neural Network

Idea of NN

Super curve fitting machine through combination of different weight and bias of a number of Activation function

Components of NN

Input nodes

Hidden layers(decided before performing NN)

 Activation function (sigmoid, Relu, soft plus)

 softplus: $f(x) = ln ( 1+ e^x)$

 sigmoid curve: $f(x) = \frac{e^x}{e^x + 1}$

 Relu: $f(x) = max(0,x)$

Output nodes

How to determine weight and bias - back-propagation

core idea: Sum of Squared Error(SSR), Gradient Descent, Chain Rule

Q: What is the target for fitting and obtaining weight & bias: minimize SSR

A: Gradient Descent(calculating derivative and step toward minimum of derivative) [of course we can calculate a derivative spectrum, but that cost too much time and calculation.]

 e.g. when only one bias b unknown, we calculate $\frac{d SSR }{ d b}$

Q: How to calculate $\frac{d SSR }{ d b}$

A: SSR = sum of (observed - predicted)^2

 predicted = weight * input + b

 therefore, we can apply Chain Rule

 $\frac{d SSR }{ d b} = \frac{d SSR }{ d predicted} \frac{d predicted }{ d b}$

 where, d SSR / d predicted = -2 * (observed - predicted), d predicted / d b = 1

 so, d SSR / d b = -2 (observed - predicted) 1

 similar approach can be applied to calculate weight.
